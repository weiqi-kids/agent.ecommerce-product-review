## 共用寫作指南

### 品牌語氣（來源：0-positioning）

| 維度 | 規格 | 範例 |
|------|------|------|
| 正式度 | 中性偏專業 | 「分析顯示」而非「我們發現」 |
| 專業度 | 白話為主，必要時用術語 | 「負評率」而非「negative sentiment ratio」 |
| 情感 | 理性、客觀 | 不使用「震驚」「難以置信」等煽情詞 |
| 人稱 | 「我們」或直接陳述 | 「本報告分析了...」 |

### 共用用語規範

| 使用 | 避免 |
|------|------|
| 揭露、分析、發現 | 爆料、踢爆、黑幕 |
| 問題、風險、注意 | 地雷、雷品、災難 |
| 根據評論分析 | 證實、確認（暗示絕對性） |
| 建議、考慮、評估 | 絕對要、千萬不要 |
| 數據顯示、評論提及 | 大家都說、網友表示 |

### 共用格式規範

| 項目 | 規格 |
|------|------|
| 段落長度 | 最多 4-5 行 |
| 句子長度 | 最多 40 字 |
| 列表使用 | 超過 3 項時使用列表 |
| 強調方式 | 粗體（重要數據）、引用區塊（評論引述） |
| 數字呈現 | 超過千用「K」，百分比保留一位小數 |

| 項目 | 規格 |
|------|------|
| 目的 | 3 秒內傳達核心價值，吸引用戶繼續瀏覽 |
| 位置 | 頁面頂部，首屏可見 |
| 內容類型 | 文字 + CTA 按鈕 |

##### 內容規格

| 元素 | 規格 | 範例/說明 |
|------|------|-----------|
| 主標題 | 10-15 字 | 「Amazon 星級看不到的真相」 |
| 副標題 | 20-30 字 | 「AI 分析數千則真實評論，揭露產品的隱藏問題」 |

| 背景 | 純色或漸層 | 避免圖片干擾閱讀 |

##### 文案方向

- 語氣：權威、直接、不煽情
- 關鍵字：必須包含「Amazon」「評論」「分析」
- 避免：「踢爆」「黑幕」「驚爆」等煽情詞

##### 參考文案選項

| 選項 | 主標題 | 副標題 |
|------|--------|--------|
| A | Amazon 星級看不到的真相 | AI 深度分析數千則評論，找出星級背後的產品問題 |
| B | 別被五星評價騙了 | 我們分析真實評論，揭露高評分產品的隱藏風險 |
| C | 買前必看的真相報告 | 從 Amazon 評論中萃取關鍵洞察，避開地雷產品 |

**建議：選項 A**（直接點出價值，不過度煽情）

##### 流程內容

```

   - 綜合分析結果
```

##### 限制內容

1. **評論可能被操控**：我們盡力識別異常模式，但無法保證所有評論都是真實的
2. **樣本可能不足**：新產品或小眾產品的評論數量可能有限
3. **時效性**：產品品質可能隨批次變化，報告反映的是分析當時的狀況
4. **不構成購買建議**：報告僅供參考，最終購買決策由用戶自行判斷
