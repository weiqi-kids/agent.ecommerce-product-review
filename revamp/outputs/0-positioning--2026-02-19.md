---

## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **網站名稱** | 買前必看 |
| **一句話描述** | AI 驅動的電商評論分析平台，揭露星級評分背後的真實問題 |
| **核心價值主張** | 從數千則 Amazon 真實評論中萃取關鍵洞察，幫助消費者避開地雷、做出明智購買決策 |

---

## 2. 目標受眾

### 主要受眾 (Primary)

| 項目 | 描述 |
|------|------|
| **是誰** | Amazon 購物者，尤其是曾被高星評分誤導而踩雷的消費者 |
| **目標/需求** | 在購買前了解產品的真實優缺點，避免買到有問題的商品 |
| **痛點** | 1. 星級評分不可靠（刷評、選擇性展示）<br>2. 評論太多無法逐一閱讀<br>3. 無法判斷哪些負評是普遍問題 |
| **使用場景** | 準備在 Amazon 下單前，搜尋「產品名 + 評價」或「產品名 + 問題」 |

### 次要受眾 (Secondary)

| 項目 | 描述 |
|------|------|
| **是誰** | 產品研究者、競品分析人員、品牌方 |
| **目標/需求** | 了解市場上同類產品的常見問題和消費者痛點 |
| **痛點** | 需要快速了解某品類的整體評價趨勢 |
| **使用場景** | 進行市場調研、產品開發決策 |

---

## 3. 差異化定位

| 項目 | 內容 |
|------|------|
| **與競品的差異** | 1. 專注於「問題揭露」而非「推薦導購」<br>2. AI 自動化分析，非人工主觀評測<br>3. 多來源整合（Amazon 評論 + Reddit + 專業評測） |
| **獨特優勢** | 1. 負評三分類系統（功能失效/副作用/物流問題）<br>2. 假貨風險偵測<br>3. 持續監控追蹤（watchlist） |
| **不做什麼** | 1. 不接受廠商合作或置入<br>2. 不做聯盟行銷導購<br>3. 不推薦未經評論驗證的產品 |

## 7. 現況觀察

### 優勢
1. 內容定位明確（問題導向，非推薦導向）
2. 報告分類清晰（警告/假貨/痛點/推薦/比較）
3. 技術基礎完善（VitePress、自動化部署）

### 待改善
1. 首頁缺乏明確的價值主張區塊
2. 報告頁缺少「相關報告」推薦
3. 沒有搜尋功能
4. 缺少用戶互動機制（評論、追蹤）
5. 品類覆蓋不均（健康保健多，電子產品少）
