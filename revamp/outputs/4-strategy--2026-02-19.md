#### 執行細節

**S01: robots.txt 修復**
- 問題：VitePress 未正確配置 public 資料夾
- 解決：在 `docs/public/` 新增 `robots.txt`
- 內容：
  ```
  User-agent: *
  Allow: /
  Sitemap: https://ecommerce.weiqi.kids/sitemap.xml
  ```

**S02: TBT 優化**
- 問題：JavaScript bundle 過大，阻塞主執行緒
- 解決方向：
  1. 程式碼分割（code splitting）
  2. 延遲載入非關鍵 JS
  3. 移除未使用的依賴

## 6. 風險與依賴

### 風險

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| VitePress 搜尋效能 | 285 頁可能影響搜尋速度 | 優先測試本地搜尋，不行再考慮 Algolia |
| TBT 優化複雜度 | 可能需要重構 VitePress 配置 | 先分析 bundle，找出最大貢獻者 |
| GitHub Pages 限制 | 無法自訂 HTTP Headers | 記錄限制，專注可控項目 |
| GitHub Traffic 限制 | 只保留 14 天數據 | 定期記錄關鍵數據，建立歷史基準 |

### 依賴

| 依賴項目 | 影響範圍 | 處理方式 |
|----------|----------|----------|
| VitePress 版本 | 搜尋功能、效能優化 | 確認當前版本支援本地搜尋 |
| GitHub Traffic | 流量指標 | 內建功能，無需設置 |
| Lighthouse CLI | 效能驗證 | 已安裝，可用 |
